# Operations runbooks

Short, practical recipes for the gallery team.

## 1 · Resolve a conflict (red cell in the Sheet)

Conflicts are flagged when ≥ 2 observation sources disagree on a field.
The auto-rules already pick a value; resolving locks it in (or overrides).

```bash
# Inspect first
$ python -m src.cli show KG-1000
$ python -m src.cli conflicts --limit 5

# Lock in the curator's value
$ python -m src.cli resolve KG-1000 classification \
    "Drawing, Collage or other Work on Paper" \
    --reason "Opaque watercolor on paper; Artsy categorises as Drawing." \
    --by sanjay@kapoors.com

# OR promote whatever the rule already chose, no value-typing needed
$ python -m src.cli promote KG-1000 classification --by sanjay@kapoors.com

# Refresh outputs and push so the Sheet picks it up
$ make consolidate report
$ git add data/human_resolutions.yaml data/master.csv data/conflicts.csv
$ git commit -m "Resolve KG-1000 classification" && git push
```

The next `Inventory ▾ → Refresh from repo` in the Sheet shows the cell
no longer red.

## 2 · Add a new auto-resolution rule

Use when the same fix would apply to many works (gallery convention).

1. Edit `data/auto_resolution_rules.yaml`. Pattern keys:
   - `<field>_contains: <substring>` — case-insensitive
   - `<field>_eq: <exact>` — exact match
   - `<field>_in: [a, b, c]` — value in list
   - `<field>_missing: true|false`
2. Run `make all` and look at the diff in `data/master.csv` /
   `data/primer_corrections.csv` to verify it fired on the right works.
3. If the change looks right, commit and push.

Example rule:

```yaml
- if:
    title_contains: thangka
  then:
    classification: Painting
  reason: "Thangkas hung on cloth are paintings on Artsy."
```

## 3 · Bulk-fix Primer from `primer_corrections.csv`

`data/primer_corrections.csv` lists 500+ specific edits to make IN
Primer to align it with the consolidated record. Each row is one
field on one work.

```
work_id, field, current_in_primer, should_be, evidence_source, reason
KG-1000, classification, Painting, Drawing, …, match_workbook, …
KG-1001, classification, Painting, Drawing, …, match_workbook, …
KG-1019, medium,         (empty),  Jade-hilted dagger…, auto_resolution, …
```

Suggested workflow:

1. Open the CSV in your spreadsheet of choice.
2. Filter to one `field` at a time (e.g. all 427 `classification`
   corrections) so you can fly through them in Primer with a single
   muscle memory.
3. For each row, find the work in Primer and apply `should_be`.
4. Mark the row done in your local copy (don't commit — the CSV is
   regenerated from sources every run).
5. When you've cleared the high-confidence corrections, run a fresh
   Primer→Artsy export, drop it in `data/raw/`, and `make all`. The
   conflict should disappear because Primer now agrees.

## 4 · Refresh from Drive

When new files arrive (a new Primer PDF, sub-inventory, photo batch):

```bash
# Drop them in data/raw/, named per `local_path:` in catalog/sources.yaml.
# Or use the bootstrap script for known sources:
$ ./scripts/bootstrap.sh

# Rebuild
$ make all

# Quick health check
$ make stats
$ make lint
$ python -m src.cli check-artsy
```

If `kg-inv lint` flags anything new, address it before pushing. Then:

```bash
$ git add data/raw/ data/master.csv data/master.json data/master_provenance.csv \
          data/conflicts.csv data/gaps.csv data/artsy_upload.csv \
          data/primer_corrections.csv reports/
$ git commit -m "Add <source>, refresh artifacts" && git push
```

CI will rerun `make all` and verify nothing broke. The Sheet picks up
the new master via **Inventory ▾ → Refresh from repo**.

## Common gotchas

- **The 108 MB Primer PDF** can't be downloaded through the in-session
  Drive MCP — use `gdown` or manual Drive download. See
  [`tools/FETCH_PRIMER_PDFS.md`](tools/FETCH_PRIMER_PDFS.md).
- **`data/inventory.db` is gitignored** — it's a derived binary.
  CI uploads it as a workflow artifact for debugging.
- **Don't edit `data/master.csv` directly** — it's regenerated by
  `make consolidate`. Edit the source-of-truth files in
  `data/raw/` or `data/auto_resolution_rules.yaml` /
  `data/human_resolutions.yaml`.
- **Tests must pass** before push. Run `make test` locally.
